# Coding Interview Prep Plan

This is a 12-week coding interview plan for easy and medium DSA problems.

The target is 100 problems in both Python and C++.

## Goals

- Build fluency in common interview patterns.
- Prioritize easy and medium problems over obscure hard problems.
- Cover arrays, strings, hashing, linked lists, trees, graphs, heaps, and DP.
- Practice clear explanations, complexity analysis, and clean implementation.
- Be able to solve each problem in both Python and C++ by the end.

## Weekly cadence

Time budget: 3 hours per week.

Recommended split:

- Session 1: learn/review pattern and solve 3 problems.
- Session 2: solve 3 problems and port key solutions to C++.
- Session 3: solve remaining problems and review mistakes.

Each problem is complete only when:

- the approach is understood,
- the Python solution passes,
- the C++ solution passes,
- time and space complexity are stated out loud,
- one edge case is tested manually.

## Review rules

Use this review loop every week:

1. Re-solve two missed problems without notes.
2. Re-implement one Python solution in C++ from memory.
3. Write the invariant or key idea for each pattern.
4. Say the time and space complexity out loud.
5. Keep a short mistake log.

## Python and C++ expectations

For Python:

- Use built-in data structures effectively.
- Know `dict`, `set`, `list`, `deque`, `heapq`, and recursion patterns.
- Prefer readable code over clever one-liners.

For C++:

- Use `vector`, `string`, `unordered_map`, `unordered_set`, `queue`, and `stack`.
- Use `priority_queue`, `deque`, `set`, `map`, and recursion when appropriate.
- Be comfortable with pointers for linked lists and trees.
- Be able to write helper functions cleanly.

## Final readiness check

You are ready for coding interviews when you can:

- solve unseen easy problems in 10-15 minutes,
- solve common medium problems in 25-35 minutes,
- explain the pattern before coding,
- debug without panic,
- translate Python solutions into C++ cleanly,
- recognize when a problem is a variant of a known pattern.

## Notes

This plan intentionally excludes hard problems.

The focus is realistic 40-minute coding interviews.
